发表于 2019-06-26 | 分类于 WEB 开发

一、视图

Django 中的视图是 一类具有相同功能和模板的网页的集合。

视图只做2件事：

  • 返回一个包含被请求页面内容的 HttpResponse 对象，或者抛出一个异常，比如 Http404 。
  • 另外一件事，随便我们怎么设计:
    • 从数据库读取记录
    • 使用一个模板引擎
    • 生成一个 PDF 文件
    • …

在 Django 中，网页和其他内容都是从视图派生而来的。每一个视图表现为一个简单的 Python 函数（或者说方法，如果是在基于类的视图里的话）。Django 会根据用户请求的 URL 中域名之后的部分来选择使用哪个视图。

为了将视图和 URl 关联起来， Django 使用了 URLconf 来配置。URLconf 将 URL 模式映射到视图。

这里只介绍 URLconf 的基础内容，你可以看看 URL 调度器 以获得更多内容。

3.2 去除模板中的硬编码

先解释两个概念，硬编码和强耦合。

硬编码：硬编码和软编码是一对组合。

  • 硬编码是指将可变变量用一个固定值来代替的方法。
  • 软编码是可以在运行时确定

前面在模板文件 polls/index.html 中，关于编码的部分就是硬链接（直接定义了一个网址 polls/question_id ）

但是假如我们想更改网址，比如改成 polls/fha/question_id ，如果使用的硬编码，你会发现需要修改 模板文件 ***.html 和 urls.py 才能完成完整的变更。
