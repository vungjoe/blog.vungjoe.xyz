二、创建模型

在 Django 里写一个数据库驱动的 Web 应用的第一步是定义模型 - 也就是数据库结构设计和附加的其它元数据。

设计哲学

模型是真实数据的简单明确的描述。它包含了储存的数据所必要的字段和行为。Django 遵循 DRY Principle（不重复地的造轮子）。它的目标是你只需要定义数据模型，然后其它的杂七杂八代码你都不用关心，它们会自动从模型生成。

修改 polls/models.py，创建两个类，并定义了类变量（数据库中的字段）

  • 每个模型被表示为 django.db.models.Model 类的子类，每个模型有一些类变量，它们都表示模型里的一个数据库字段。
  • 每个字段都是 Field 类的实例，以下是几个常见的 Field 类：
    • 字符字段：CharField
    • 日期时间字段：DateTimeField
    • 布尔字段：BooleanField
    • …
  • ForeignKey，也是一个 Field 类，只不过是较为特殊的关系 Field 类。上面这句代码的含义是——每个 Choice 对象都关联到一个 Question 对象

三、path() 函数

urlpatterns 列表中的 path() 函数，一共有四个参数，包括两个必须参数 route 和 views ，两个可选参数 kwargs 和 name 。

path(route, view, kwargs=None, name=None)

3.1 route

  • route 是一个匹配 URL 的准则（类似正则表达式）
  • 当 Django 响应一个请求时，它会从 urlpatterns 列表的第一项开始，按顺序依次匹配列表中的项，直到找到匹配的项
  • route 不会匹配 GET 和 POST 参数或域名。例如，
    • URLconf 在处理请求 https://www.example.com/myapp/ 时，它会尝试匹配 myapp/
    • 处理请求 https://www.example.com/myapp/?page=3 时，也只会尝试匹配 myapp/。

3.2 view

当 Django 找到了一个匹配的准则，就会调用这个特定的视图函数，并传入一个 HttpRequest 对象作为第一个参数，被“捕获”的参数以关键字参数的形式传入。稍后，我们会给出一个例子。

3.3 kwargs

二、基本概念

2.1 表（tables）：5个

iptables包含五张表（tables）：

  • raw：用于配置数据包，raw中的数据包不会被系统跟踪。
  • mangle：用于对特定的包做修改。
  • nat：nat就是网络地址转换。用于网络地址转换（如端口映射、地址映射等）。生活中的实例——路由器。
  • filter：默认表，用于过滤包。
  • security：用于强制访问控制网络规则。CentOS6中是没有的，CentOS7中增加。工作中应该用不到。

大多数情况下，我们只需要使用nat和filter。

2.2 链（chains）

表是由链组成的，链是一些按顺序排列的规则的列表。
默认情况下，任何链中都是没有规则的。可以根据我们自己的需要添加规则。

  • filter 表包含 INPUT、OUTPUT、FORWARD 3条内建的链，这3条链作用于数据包过滤过程中的不同时间点。
    • INPUT ：对进入本机的包进行过滤
    • OUTPUT ：对从本机送出去的包进行过滤
    • FORWARD ：不是本机的包，对目标地址做更改或者说是转发。
  • nat 表包含 PREROUTING、POSTROUTING、OUTPUT 3条链。
    • PREROUTING ：外网 IP 转内网 IP 。
    • POSTROUTING ：内网 IP 转外网 IP 。
    • OUTPUT ：修改从本机出去的包。

2.3 规则（rules）

数据包的过滤基于规则，规则由一个目标（数据包包匹配所有条件后的动作）和很多匹配（导致该规则可以应用的数据包所满足的条件）指定。一个规则的典型匹配事项是数据包进入的端口（例如：eth0 或者 eth1）、数据包的类型（ICMP, TCP, 或者 UDP ）和数据包的目的端口。

目标（怎么处理数据包）使用-j或者--jump选项指定。

2.4 遍历链（Traversing Chains）

开头的原理图描述了链在任何接口上收到的网络数据包是按照怎样的顺序穿过表的交通管制链：

  • 第一个路由策略包括决定数据包的目的地是本地主机（这种情况下，数据包穿过 INPUT 链），还是其他主机（数据包穿过 FORWARD 链）；
  • 中间的路由策略包括决定给传出的数据包使用哪个源地址、分配哪个接口；
  • 最后一个路由策略存在是因为先前的 mangle 与 nat 链可能会改变数据包的路由信息。
