各文件的用途：

  • 最外层的目录 mold ：根目录只是项目的容器，这个名字没有意义，可以任意重命名为你喜欢的名字。
  • manage.py：一个让你用各种方式管理 Django 项目的命令行工具。可阅读 django-admin and manage.py 来获取所有有关的细节。
  • 子目录 mold ：这个目录包含你的项目，它是一个纯 Python 的包。它的名字就是当你引用它内部任何东西是需要用到的 Python 包名。（例如 mold.urls ）
  • mold/__init__.py：一个空文件，告诉 Python 这个目录应该被认为是一个 Python 包。更多有关 Python 中包的知识。
  • mold/settings.py： Django 项目的配置文件。了解更多请看 Django settings。
  • mold/urls.py： Django 项目的 URL 声明，就像网站的“目录”一样。了解更多请看 URL 调度器。
  • mold/wsgi.py：作为项目的运行在 WSGI 兼容的 Web 服务器上的入口。阅读 如何使用 WSGI 进行部署 了解更多细节。

2.1 创建第一个应用

在 Django 中，每一个应用都是一个 Python 包，并且遵循相同的约定。 Django 自带一个工具，可以帮你生成应用的基础目录结构，这样就可以使你只专注于代码编写，而不是创建目录了。

项目 VS 应用

项目和应用有什么区别？应用是一个专门做某件事的网络应用程序——比如博客系统，比如公共记录的数据库，或者是简单的投票程序。项目则是一个网站使用的配置和应用的集合。

  • 项目可以包含多个应用
  • 应用可以被多个项目使用

三、path() 函数

urlpatterns 列表中的 path() 函数，一共有四个参数，包括两个必须参数 route 和 views ，两个可选参数 kwargs 和 name 。

path(route, view, kwargs=None, name=None)

3.1 route

  • route 是一个匹配 URL 的准则（类似正则表达式）
  • 当 Django 响应一个请求时，它会从 urlpatterns 列表的第一项开始，按顺序依次匹配列表中的项，直到找到匹配的项
  • route 不会匹配 GET 和 POST 参数或域名。例如，
    • URLconf 在处理请求 https://www.example.com/myapp/ 时，它会尝试匹配 myapp/
    • 处理请求 https://www.example.com/myapp/?page=3 时，也只会尝试匹配 myapp/。

3.2 view

当 Django 找到了一个匹配的准则，就会调用这个特定的视图函数，并传入一个 HttpRequest 对象作为第一个参数，被“捕获”的参数以关键字参数的形式传入。稍后，我们会给出一个例子。

3.4 name

为你的 URL 取名能使你在 Django 的任意地方唯一地引用它，尤其是在模板中。这个有用的特性允许你只改一个文件就能全局地修改某个 URL 模式。
