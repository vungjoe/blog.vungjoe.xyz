Django快速入门教程第04节——表单和通用视图

发表于 2019-06-26 | 分类于 WEB 开发

官方文档阅读：编写你的第一个 Django 应用，第 4 部分

一、表单

1.1 表单概述

Django 开发的是动态 web 服务，而非单纯地提供静态页面。动态服务的本质在于和用户进行互动，接受用户的输入，根据输入的不同，返回不同的内容给用户。

  • 返回数据是服务器后端做的
  • 接收用户输入的就需要靠 HTML 表单

表单 <form>...</form> 可以收集其内部标签中的用户输入，然后将数据发送到服务器。

一个 HTML 表单必须包含两个内容：

  • 目的地：用户数据发送的目的 URL
  • 方式：发送数据做使用的 HTML 方法。

处理表单时只会用到 POST 和 GET 方法：

  • GET 方法：GET 方法将用户数据以 键=值 的形式，用 & 组合在一起成为一个整体字符串，最后添加前缀 ?，将字符串拼接到 URL 内，生成一个类似 https://docs.djangoproject.com/search/?q=forms&release=1 的 URL
  • POST 方法：浏览器会组合表单数据、对它们进行编码，然后打包将其发送到服务器，数据不会出现在 URL 中
  • GET 方法通常用来请求数据，不适合密码表单这一类保密信息的发送，也不适合数据量大的表单和二进制数据。对于这类数据应该使用 POST 方法
  • GET 适合网页搜索的表单，因为这种表示一个 GET 请求的 URL 可以很容易地设置书签、分享和重新提交。

Django 在内部集成了一个表单模块，专门帮我们处理表单相关的内容（不需要我们手动编写 HTML 表单）。Django 表单模块提供了三个主要功能：

  • 准备和重构数据用于页面渲染
  • 为数据创建 HTML 表单元素
  • 接收和处理用户从表单发送过来的数据

二、通用视图

在前面编写的视图文件中，仔细观察会发现，关于 result 、 detail 的部分几乎一模一样，这样就存在一个问题——代码冗余。此外，关于 index 的部分也和这两个类似。

这几个视图反映了基本的 web 开发中的一个常见情况：根据 URL 的参数来获取数据库中的数据 → 载入模板文件 → 返回渲染后的模板。

基于此， Django 提供了一个快捷方式，叫做“通用视图”系统。

如果已经编写的视图未使用通用视图，则需要经过以下三个步骤来完成转换：

  1. 转换 URL conf
  2. 删除旧的、不再需要的视图
  3. 基于 Django 的通用视图引入新视图
